# REMOTETABLE — Веб-интерфейс
## Полная инструкция по установке и запуску

---

## ШАГ 1. Что нужно установить на компьютер

### Установите Node.js
Скачайте с официального сайта: **https://nodejs.org**
- Нужна версия **18 или выше** (рекомендуется 20 LTS)
- При установке галочка "Add to PATH" должна стоять

Проверьте в терминале (Командная строка / PowerShell):
```
node --version
```
Должно показать `v18.x.x` или `v20.x.x`

### Установите Bun
Bun — это быстрый менеджер пакетов (заменяет npm).

**Windows (PowerPanel):**
```
powershell -c "irm bun.sh/install.ps1 | iex"
```

**Linux / macOS:**
```
curl -fsSL https://bun.sh/install | bash
```

Проверьте:
```
bun --version
```

## ШАГ 2. Распакуйте архив

Распакуйте скачанный архив `remotable-web.zip` в любую папку.
Например, прямо в корень вашего проекта REMOTETABLE:

```
REMOTETABLE/
├── web/                    ← СЮДА распакуйте архив
│   ├── src/
│   ├── prisma/
│   ├── db/
│   ├── mini-services/
│   ├── package.json
│   └── ...
├── client/
├── specialist/
├── core/
├── network/
└── ...
```

---

## ШАГ 3. Установите зависимости

Откройте терминал и перейдите в папку с распакованным проектом:

```bash
cd web
```

Установите зависимости основной части:
```bash
bun install
```

Установите зависимости чат-сервиса:
```bash
cd mini-services/chat-service
bun install
cd ../..
```

> Это скачает все нужные библиотеки (~200 МБ). Подождите завершения.

---

## ШАГ 4. Инициализация базы данных

База данных (SQLite) уже включена в архив в папке `db/`.
Если она не работает, пересоздайте:

```bash
bun run db:push
```

---

## ШАГ 5. Запуск

Нужно открыть **2 терминала** (2 окна командной строки).

### Терминал 1 — Основной сервер (веб-интерфейс)
```bash
cd web
bun run dev
```
Дождитесь сообщения: `✓ Ready in XXXms`

### Терминал 2 — Чат-сервис (для обмена сообщениями)
```bash
cd web
bun run chat
```

### Откройте в браузере
```
http://localhost:3000
```

## Как пользоваться

### Регистрация
1. Откройте `http://localhost:3000`
2. Нажмите **"Зарегистрироваться"**
3. Введите логин, пароль, выберите роль:
   - **Пользователь** — тот, кому нужна помощь
   - **Специалист** — тот, кто помогает

### Для пользователя
1. Войдите под учётной записью пользователя
2. Нажмите **"Создать заявку"**
3. Заполните форму: категория, приоритет, описание проблемы
4. Нажмите **"Создать"**
5. Дождитесь, пока специалист примет заявку
6. Общайтесь со специалистом в чате

### Для специалиста
1. Войдите под учётной записью специалиста
2. На дашборде увидите очередь заявок
3. Нажмите **"Принять"** рядом с нужной заявкой
4. Откройте заявку и общайтесь с пользователем в чате
5. Нажмите **"Начать сеанс"** для удалённого подключения
6. После решения проблемы — **"Завершить"**

## Возможные проблемы

### "Порт 3000 уже занят"
```bash
# Windows (PowerShell)
Get-Process -Id (Get-NetTCPConnection -LocalPort 3000).OwningProcess | Stop-Process -Force

# Linux / macOS
lsof -i :3000
kill -9 <PID>
```

### "Порт 3004 уже занят" (чат)
```bash
# Windows
Get-Process -Id (Get-NetTCPConnection -LocalPort 3004).OwningProcess | Stop-Process -Force

# Linux / macOS
lsof -i :3004
kill -9 <PID>
```

### "bun: command not found"
Bun не установлен или не добавлен в PATH. Переустановите по инструкции из Шага 1.

### "prisma db push" ошибка
```bash
cd web
mkdir -p db
bun run db:push
```

### Белый экран / ошибка при открытии
Убедитесь, что запущены ОБА сервиса:
- `bun run dev` (Терминал 1) — веб-сервер
- `bun run chat` (Терминал 2) — чат-сервис
