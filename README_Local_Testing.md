# Локальное тестирование системы удалённой техподдержки

## Быстрый старт

### 1. Установка зависимостей

```bash
# Установка всех зависимостей одной командой
pip install -r requirements.txt
```

### 2. Настройка PostgreSQL

**Windows:**
- Скачайте PostgreSQL с https://www.postgresql.org/download/windows/
- Установите с паролем `postgres`
- Добавьте PostgreSQL в PATH

**Linux:**
```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
sudo systemctl start postgresql
sudo -u postgres psql -c "ALTER USER postgres PASSWORD 'postgres';"
```

**macOS:**
```bash
brew install postgresql
brew services start postgresql
```

### 3. Создание базы данных

```bash
# Войдите в psql
psql -U postgres

# Создайте базу данных
CREATE DATABASE remote_desktop;
\q
```

### 4. Настройка переменных окружения

Создайте файл `.env` в корневой папке проекта:

```bash
DB_HOST=localhost
DB_PORT=5432
DB_NAME=remote_desktop
DB_USER=postgres
DB_PASSWORD=postgres

RELAY_HOST=localhost
RELAY_PORT=6969
```

Или запустите автоматическую настройку:

```bash
python setup_local.py
```

## Запуск компонентов

Откройте 3 отдельных терминала:

### Терминал 1: Реле-сервер (обязателен для NAT traversal)

```bash
python relay_server.py
```

Ожидаемый вывод:
```
╔══════════════════════════════════════════════════╗
║        РЕЛЕ-СЕРВЕР УДАЛЁННОГО ДОСТУПА           ║
╠══════════════════════════════════════════════════╣
║  Хост: localhost                               ║
║  Порт: 6969                                    ║
╚══════════════════════════════════════════════════╝

[RELAY] Реле-сервер запущен на порту 6969
```

### Терминал 2: Сервер специалиста (ожидает клиента)

```bash
python server.py
```

Ожидаемый вывод:
```
Запуск сервера удалённого доступа...
Подключение к localhost:6969...
Подключено к реле-серверу
Ожидание подключения клиента...
```

### Терминал 3: Клиент (подключается к специалисту)

```bash
python client.py
```

Ожидаемый вывод:
```
Запуск клиента удалённого доступа...
Подключение к localhost:6969...
Подключено к реле-серверу
Ожидание подтверждения подключения...
```

## Альтернативный запуск через GUI

### Терминал 1: Реле-сервер

```bash
python relay_server.py
```

### Терминал 2: GUI для клиента

```bash
python auth_gui.py
```

Откроется окно регистрации/входа. После входа можно запустить режим ожидания подключения.

## Порядок тестирования

### Тест 1: Базовое соединение

1. Запустите `relay_server.py`
2. Запустите `server.py` (специалист)
3. Запустите `client.py` (клиент)
4. На клиенте появится окно с запросом доступа
5. Нажмите "Разрешить"
6. У специалиста появится окно просмотра экрана клиента

### Тест 2: Управление мышью

1. В окне специалиста кликните мышью на экране клиента
2. Убедитесь что курсор двигается на компьютере клиента

### Тест 3: Передача клавиш

1. Нажмите клавишу (например, 'a')
2. Убедитесь что символ появляется в активном поле ввода на клиенте

### Тест 4: Разрыв соединения

1. Нажмите 'q' в окне специалиста
2. Соединение должно корректно закрыться

## Возможные проблемы и решения

### Ошибка: "Port already in use"

```bash
# Найдите процесс использующий порт 6969
netstat -ano | findstr :6969    # Windows
lsof -i :6969                    # Linux/macOS

# Убейте процесс или измените порт в конфигурации
```

### Ошибка: "could not connect to database"

```bash
# Проверьте что PostgreSQL запущен
sudo systemctl status postgresql   # Linux
brew services list                # macOS

# Или запустите службу
sudo systemctl start postgresql   # Linux
brew services start postgresql    # macOS
```

### Ошибка: "No module named 'cv2'"

```bash
pip install opencv-python-headless
```

### Клиент не подключается к специалисту

1. Проверьте что оба используют один `RELAY_HOST` (localhost для локальной сети)
2. Проверьте логи на предмет ошибок
3. Убедитесь что брандмауэр не блокирует порт 6969

### Окно специалиста не показывает изображение

1. Проверьте что клиент отправил разрешение экрана
2. Проверьте консоль клиента на ошибки
3. Убедитесь что `pyautogui` работает корректно

## Структура логов

Приложения создают логи в текущей директории:

- `client.log` - логи клиента
- `server.log` - логи сервера специалиста
- `relay_server.log` - логи реле-сервера

Для просмотра логов в реальном времени:

```bash
# Windows
type client.log

# Linux/macOS
cat client.log
```

## Конфигурация для разных сценариев

### Локальная сеть (несколько компьютеров)

Замените `localhost` на IP-адрес компьютера с реле-сервером:

```bash
RELAY_HOST=192.168.1.100  # IP компьютера с relay_server.py
```

### Тестирование на одном компьютере

Всё работает на `localhost` - просто запустите все компоненты в разных терминалах.

## Команды для очистки

```bash
# Удаление базы данных (пересоздание)
sudo -u postgres psql -c "DROP DATABASE remote_desktop;"
sudo -u postgres psql -c "CREATE DATABASE remote_desktop;"

# Очистка логов
rm -f *.log
```
